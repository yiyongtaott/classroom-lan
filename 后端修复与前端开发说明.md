## 一、后端问题修复

### 问题诊断
原后端 WebSocket 使用 `com.sun.net.httpserver` + 手动帧编码（`UserWsHandler.java`）：

- **非标准** - 手动构造 WebSocket 帧（0x81 标记等），缺少完整协议处理（掩码、ping/pong、帧分片）
- **连接生命周期异常** - `Thread.sleep(5000)` 后强制断开（线量占位符，仅演示用）
- **协议不完整** - 不支持持久连接、多路复用等特性
- **无标准注解** - 无法使用 `@ServerEndpoint`、`@OnMessage` 等 JSR-356 注解

🔧 **参考实现** (`参考/WsServer.java`, `参考/ChatWsEndpoint.java` 等) 使用 **Tyrus** (Jakarta WebSocket Reference Implementation) 标准注解端点。

### 修复方案

**1. 添加 Tyrus 依赖** (`backend/pom.xml`)

```xml
<dependency>
    <groupId>org.glassfish.tyrus</groupId>
    <artifactId>tyrus-server</artifactId>
    <version>2.2.2</version>
</dependency>
<dependency>
    <groupId>org.glassfish.tyrus</groupId>
    <artifactId>tyrus-container-grizzly-server</artifactId>
    <version>2.2.2</version>
</dependency>
<dependency>
    <groupId>jakarta.websocket</groupId>
    <artifactId>jakarta.websocket-api</artifactId>
    <version>2.1.1</version>
</dependency>
```

**2. 创建标准 WebSocket 端点** (`backend/src/main/java/io/classroomlan/server/ws/`)

| 文件 | 路径 | 说明 |
|------|------|------|
| `ChatWsEndpoint.java` | `/ws/chat` | 群聊 — 历史消息、广播 |
| `GameWsEndpoint.java` | `/ws/game` | 游戏房间 — 玩家进出、状态同步 |
| `UserWsEndpoint.java` | `/ws/user` | 用户在线列表广播 |

全部使用 `@ServerEndpoint` + `@OnOpen/@OnMessage/@OnClose/@OnError` 注解，线程安全使用 `ConcurrentHashMap`。

**3. 更新 `WsServer.java`**

使用 Tyrus 内嵌 Grizzly：
```java
server = new Server("0.0.0.0", 8081, "/", null,
    GameWsEndpoint.class, ChatWsEndpoint.class, UserWsEndpoint.class);
server.start();
```

**4. 修正 `HttpServer.java`**

- 删除冲突的 `/ws/*` 上下文（WebSocket 由 `WsServer` 独占端口 8081）
- `/api/chat/*` 改为 REST 备份接口 (`ChatHttpHandler.java`)
  - `GET /api/chat/history` — 历史消息
  - `GET /api/chat/online` — 在线统计
  - `POST /api/chat/send` — 发送消息（供 REST 调用）

**5. 精简 Handler 集合**

移除非必要 / 与参考不符的旧类：
- ❌ `ChatHandler.java` (仅返回 info 的占位类)
- ❌ `UserWsHandler.java` (自定义帧编码, 5s 自动断开)

保留：
- ✅ `StaticHandler`, `ApiFileHandler`, `StatusHandler`, `UserHandler`

**6. `pom.xml` 完整配置**

- Java 17 编译
- Shade plugin 打包 fat JAR（含 Tyrus 依赖 + `ServicesResourceTransformer` 合并 SPI）
- 过滤签名文件避免 SecurityException

## 二、Vue 3 前端实现

### 技术栈

| 工具 | 版本 | 用途 |
|------|------|------|
| Vue | 3.x | UI 框架 |
| Vue Router | 4.x | 路由 (hash mode) |
| Pinia | 2.x | 状态管理 |
| Vite | 5.x | 构建工具 |

### 文件结构

```
frontend/
├── package.json
├── vite.config.js          # 构建输出到后端 resources/static/
├── index.html
└── src/
    ├── main.js             # 应用入口
    ├── App.vue             # 根组件 (全局布局)
    ├── router/index.js     # hash 路由
    ├── stores/
    │   ├── node.js         # 节点状态 (role/peerCount)
    │   ├── chat.js         # 聊天消息 + 在线用户
    │   ├── game.js         # 房间 + 游戏状态
    │   └── UserStore.js    # 用户 stub
    ├── composables/
    │   └── useWebSocket.js # 可复用 WS 连接 + 消息路由
    └── views/
        ├── Home.vue        # 首页 + 节点状态卡片
        ├── FileTransfer.vue
        ├── GameLobby.vue   # 房间列表 + 创建/加入
        └── games/
            ├── DrawGuess.vue    # Canvas 绘图 + 猜测
            ├── Quiz.vue         # 四选项答题 + 倒计时
            └── Werewolf.vue     # 角色揭示 + 投票流程
```

### 核心功能

#### 1. WebSocket 连接 (`useWebSocket.js`)

- 单例模式，同一 URL 复用连接
- 自动重连（3s 重试，最多 5 次）
- `onMessage(handler)` 注册回调
- 发送队列：连接就绪前先缓存消息

```js
const ws = useWebSocket('/ws/chat')
ws.connect()
ws.onMessage(data => console.log(data))
ws.send({ action: 'join', nickname: name })
```

#### 2. 状态管理 (Pinia)

- **nodeStore** — 轮询 `/api/status` 更新 `role`（`LEADER`/`FOLLOWER`/`CANDIDATE`）
- **chatStore** — WebSocket 消息入队，保留最新的 100 条
- **gameStore** — 当前房间 ID + 玩家列表

#### 3. 游戏页面逻辑

| 游戏 | 核心交互 | WebSocket 事件 |
|------|---------|----------------|
| **你画我猜** | Canvas 绘图 (笔画实时同步预留)，猜词输入 | `GAME_ACTION` (STROKE / HINT / GUESS) |
| **快问快答** | 点击选项按钮，15s 倒计时 | `NEW_QUESTION` / `ANSWER_RESULT` |
| **谁是卧底** | 角色揭示 → 描述 → 投票 | `ROLE_REVEAL` / `DESCRIPTION` / `VOTE` |

## 三、API 对照表

### HTTP (端口 8080)

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/` | 返回 `index.html` (SPA) |
| `GET` | `/api/status` | `{"role":"LEADER","selfIp":"...","peers":3}` |
| `GET` | `/api/peers` | `[{"nodeId":"..."}]` |
| `GET` | `/api/files` | 文件列表 `[{"id","name","size","uploadedBy"}]` |
| `POST` | `/api/upload` | multipart/form-data 上传 |
| `GET` | `/api/download/{id}` | 文件下载 |

### WebSocket (端口 8081)

| 端点 | 消息方向 | 示例 |
|------|---------|------|
| `/ws/chat` | C→S: `{action:"join|send|ping"}`<br>S→C: `{type:"message|history|pong"}` | 群聊广播 |
| `/ws/game` | C→S: `{action:"JOIN_ROOM|LEAVE_ROOM|GAME_ACTION"}`<br>S→C: `{event:"ROOM_STATE\|GAME_STATE\|PLAYER_JOINED\|ERROR"}` | 游戏房间控制 |
| `/ws/user` | C→S: `{action:"register|ping"}`<br>S→C: `{type:"welcome\|userList\|pong"}` | 在线用户列表 |

## 四、构建与运行

### 前置

- JDK 17
- Maven 3.8+
- Node.js 18+

### 前端构建

```bash
cd frontend
npm install
npm run build   # 输出 → backend/src/main/resources/static/
```

### 后端打包

```bash
cd backend
mvn package   # 生成 fat JAR: target/classroomlan-1.0.0.jar
```

### 运行（开发）

```bash
cd backend
mvn exec:java -Dexec.mainClass=io.classroomlan.Main
```

然后浏览器打开 `http://localhost:8080`

## 六、注意事项

1. **单 Leader 原则** — 同一 LAN 仅有一个节点运行 HTTP+WS，其余 Follower 只作为浏览器客户端
2. **无持久化** — 文件存储在 `java.io.tmpdir/classroomlan/`，重启丢失
3. **Windows 防火墙** — 首次运行需放行 UDP 9999 / TCP 8080/8081
4. **前端缓存** — `node` store 轮询 `/api/status` 获取最新角色；WebSocket 负责实时
5. **端口冲突** — 8080/8081 被占用时当前代码不会自动切换（TODO: 参考文档里的 port retry 逻辑）
